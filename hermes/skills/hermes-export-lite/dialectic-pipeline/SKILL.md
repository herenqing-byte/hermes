# Dialectic Analysis Pipeline

三阶段辩证分析：**Draft → Critique → Synthesis**。适用于任何需要深度、多视角分析的内容。

## 执行模式选择

- **简单文档**（<3k 字 / 单一主题）：单 session 内顺序完成 3 个 Phase
- **复杂报告**（>3k 字 / 多主题 / 数据密集）：spawn 独立 sub-agent 分别做 Phase 1 和 Phase 2，主 Agent 做 Phase 3 综合裁判
- 编排示例见 `scripts/run-pipeline.sh`

## Phase 1: Draft（深度分析）

**角色**：领域专家 — 形成有立场的分析，拒绝"两边都有道理"的太极拳。

**步骤**：
1. 通读全文，识别核心议题
2. 对每个议题形成明确立场
3. 按以下结构输出

**输出格式**：

```
## 📝 Draft 分析

### 核心论断
- [论断1]：[一句话立场]
- [论断2]：...

### 支撑证据
| 论断 | 证据 | 证据强度(强/中/弱) |
|------|------|---------------------|
| ...  | ...  | ...                 |

### 趋势判断
- [趋势1]：方向 + 时间窗口 + 依据

### 行动建议
1. **立即**：...
2. **短期（1-3月）**：...
3. **长期（3月+）**：...
```

**红线**：每个论断必须可证伪（能说出"如果X发生则论断错误"）。模糊论断打回重写。

## Phase 2: Critique（对抗性质疑）

**角色**：批判审查员 — 目标是击穿 Draft，不是温柔建议。

**步骤**：
1. 逐条审查 Draft 的论断和证据
2. 按三色分级质疑（详见 `references/critique-checklist.md`）
3. 至少找出 1 个 🔴 红色质疑；找不到则明确说明原因

**质疑分级**：
- 🔴 **红色**：逻辑硬伤、数据造假/误用、关键遗漏 — 足以推翻结论
- 🟡 **黄色**：推理跳跃、样本偏差、立场偏见 — 削弱但不推翻
- 🟢 **绿色**：可改进但不影响核心结论

**输出格式**：

```
## 🔍 Critique 质疑报告

### 🔴 红色质疑
1. **[质疑标题]**
   - 针对论断：[Draft 中的哪条]
   - 问题：[具体描述]
   - 反例/反证：[如果存在]
   - 杀伤力：[如何推翻该结论]

### 🟡 黄色质疑
1. **[质疑标题]**
   - 针对论断：...
   - 问题：...
   - 改进方向：...

### 🟢 绿色质疑
1. ...

### 红色质疑缺失说明（如无红色质疑）
- 为什么 Draft 在逻辑和数据层面站得住：...
```

## Phase 3: Synthesis（综合裁判）

**角色**：最终决策者 — 综合 Draft 和 Critique，输出可行动的结论。

**步骤**：
1. 逐条比对 Draft 论断 vs Critique 质疑
2. 判定每条结论的存活状态
3. 标注置信度，识别信息缺口

**输出格式**：

```
## ⚖️ Synthesis 综合裁决

### 结论存活表
| Draft 论断 | Critique 质疑 | 裁决 | 置信度 |
|-----------|--------------|------|--------|
| [论断1]   | [对应质疑]    | ✅ 成立 / ⚠️ 需修正 / ❌ 推翻 | 高/中/低 |

### 修正后结论
- [经受住质疑的结论，原文保留]
- [需修正的结论，给出修正版本及理由]

### 最终建议
1. **[建议1]** — 置信度：高 | 依据：...
2. **[建议2]** — 置信度：中 | 依据：...
3. **[建议3]** — 置信度：低 | 依据：...

### 信息缺口
- [还需要什么数据/验证才能将"中/低"提升为"高"]
```

## Sub-Agent 编排（复杂报告）

spawn 时 task description 模板：

**Phase 1 sub-agent**：
```
你是领域专家。阅读以下内容，按 Draft 格式输出深度分析。
要求：有明确立场，不做太极拳。每个论断必须可证伪。
[贴入待分析内容或文件路径]
```

**Phase 2 sub-agent**：
```
你是批判审查员。阅读以下 Draft 分析，按 Critique 格式输出质疑报告。
要求：至少 1 个红色质疑，找不到要说明原因。
参考质疑清单：references/critique-checklist.md
[贴入 Phase 1 Draft 输出]
```

主 Agent 收到两个输出后，自行完成 Phase 3 Synthesis。
