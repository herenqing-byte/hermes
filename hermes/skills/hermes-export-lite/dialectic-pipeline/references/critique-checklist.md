# Critique 质疑清单

Phase 2 审查时逐项过一遍。不需要每条都命中，但每条都要考虑。

## 🔴 红色质疑（足以推翻结论）

| # | 维度 | 检查问题 |
|---|------|----------|
| R1 | 逻辑谬误 | 是否存在循环论证、稻草人、滑坡谬误、虚假二分？ |
| R2 | 数据伪造/误用 | 数据来源是否可靠？是否 cherry-pick 了有利数据？ |
| R3 | 因果倒置 | 是否把相关性当因果性？有无混淆变量？ |
| R4 | 关键遗漏 | 是否忽略了明显的反面证据或竞争假说？ |
| R5 | 幸存者偏差 | 结论是否只基于成功案例而忽略失败案例？ |
| R6 | 定义滑移 | 关键概念在论证中是否偷换了定义？ |

## 🟡 黄色质疑（削弱但不推翻）

| # | 维度 | 检查问题 |
|---|------|----------|
| Y1 | 推理跳跃 | 从证据到结论之间是否存在未论证的飞跃？ |
| Y2 | 样本偏差 | 样本是否具代表性？时间窗口是否太短？ |
| Y3 | 立场偏见 | 分析者是否有利益相关导致偏向性？ |
| Y4 | 过度泛化 | 是否从局部案例推广到整体？ |
| Y5 | 忽略反馈环 | 是否考虑了行动本身对系统的反馈影响？ |
| Y6 | 时效性 | 依据的数据/趋势是否已过时？ |

## 🟢 绿色质疑（可改进）

| # | 维度 | 检查问题 |
|---|------|----------|
| G1 | 表述精度 | 结论是否足够具体可执行？ |
| G2 | 替代方案 | 是否探索了足够的替代路径？ |
| G3 | 边界条件 | 结论在什么条件下不成立？是否说明了？ |
| G4 | 可验证性 | 建议是否附带了验证方法和时间节点？ |

## 使用方法

1. 读完 Draft 后，逐条过清单
2. 命中的项标记对应颜色和编号（如 R2、Y3）
3. 对每个命中项写出：针对哪条论断 → 具体问题 → 反例或改进方向
4. 红色质疑必须有 ≥1 条，或明确说明为什么 R1-R6 均不命中
