# Hypothesis Tracker

管理活跃假设库，让每篇材料的分析不再是孤岛。

## 假设库位置

默认：`/Users/herenqing/.hermes/workspaces/openclaw-migration/research/trend-system/trend-hypotheses.md`
初始化模板：读取 `references/hypothesis-template.md`，复制到默认路径。

## 假设格式

每条假设包含：

| 字段 | 说明 |
|------|------|
| ID | H-001 格式，自增 |
| 陈述 | 一句话趋势判断 |
| 置信度 | 高 / 中 / 低 |
| 状态 | 活跃 / 已验证 / 已否决 / 已合并 |
| 支撑证据 | 列表，每条含摘要 + 来源链接 + 日期 |
| 反驳证据 | 同上 |
| 创建时间 | YYYY-MM-DD |
| 最后更新 | YYYY-MM-DD |

## 五种操作及判断标准

### 1. 新增
**触发**：材料中出现尚无对应假设的趋势判断，且有至少一条具体证据支撑。
- 不要为观点碎片建假设——必须是可验证的趋势判断。
- 初始置信度：有 2+ 独立来源 → 中；仅 1 来源 → 低。

### 2. 强化
**触发**：新证据从**独立来源**支持现有假设的核心论断。
- 仅当新证据提供了**增量信息**（新数据点、新角度、新参与者）时追加。
- 同一来源的重复报道不算强化。
- 置信度提升规则：低→中 需 3+ 条独立支撑；中→高 需 5+ 条且跨 2+ 来源类型（研报/新闻/产品动态/数据）。

### 3. 挑战
**触发**：新证据与假设核心论断**直接矛盾**，或关键前提被削弱。
- 不轻易挑战——仅记录与假设直接相关的反面证据。
- 置信度降级规则：每 2 条高质量反驳证据降一级。

### 4. 合并
**触发**：两个假设的核心论断实质相同，仅措辞/角度不同。
- 保留证据更丰富的那条为主，另一条标记"已合并→H-XXX"。
- 合并双方证据。

### 5. 否决
**触发**：满足以下任一条件：
- 3+ 条高质量反驳且无有效支撑回应
- 假设的关键前提被事实证伪（如公司已关闭相关业务）
- 标记状态为"已否决"，记录否决原因，**不删除**。

## 材料匹配流程

读完一篇材料后，执行：

1. **读取假设库**（仅活跃状态）
2. **提取材料关键论断**——列出 3-7 条核心论断
3. **逐一匹配**：每条论断 × 每条活跃假设，判断"支持/反驳/无关"
4. **执行操作**：按上述标准执行强化/挑战
5. **检查新假设**：未匹配任何现有假设的重要论断 → 考虑新增
6. **输出报告**（简洁格式）：

```
## 假设匹配报告
**材料**：[标题](链接)

**强化**：
- H-003（置信度 中→中）：新增证据「...」

**挑战**：
- H-007（置信度 中→低）：反驳证据「...」

**新增**：
- H-012：「...」（置信度：低）

**无变化**：H-001, H-002, H-005 ...
```

## 假设库维护

- **周报/月报场景**：生成前先读假设库，以活跃假设为分析框架组织内容。
- **季度审查**：检查所有"低置信度"假设，超过 60 天无新证据 → 标记否决或归档。
- 假设库文件用 Markdown 表格做索引 + `<details>` 折叠块存详情，格式见模板。
