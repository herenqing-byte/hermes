# 文档骨架提取

从任意文档中提取结构化骨架，将隐性分析显性化。

## 流程

1. **获取全文** — 链接用 web_fetch/browser/KM skill 读取；粘贴文本直接用
2. **长文预处理** — >5k 字时先分段扫描，标记各段主旨，再合成骨架
3. **输出骨架** — 严格使用下方固定格式，不增删层级
4. **追问** — 用户可要求展开任一层的细节

## 输出格式（固定，逐字遵循）

```
## 🎯 核心论断

（1-3 句。这篇文档到底在说什么？如果只能记住一件事，是什么？）

## 📊 关键证据

1. ✅/⚠️/❌ （证据描述）
2. ...
（3-5 条。✅ 强证据=可验证可复现 | ⚠️ 弱证据=单一来源/推断 | ❌ 无证据=纯观点）

## 🕳️ 信息缺口

1. （该有但没出现的内容 / 作者回避的问题 / 做决策还缺的信息）
（1-3 条。没有则写"未发现明显缺口"）

## ⚡ 决策点

1. （需要做什么判断？不做的代价？有截止时间吗？）
（0-3 条。纯知识性文档可写"无需即时决策"）
```

## 规则

- **证据标注必填** — 每条证据必须带 ✅/⚠️/❌ 前缀，这是本 skill 的核心价值
- **克制** — 骨架就是骨架，不填肉。每层不超过规定条数
- **立场透明** — 如果作者有明显立场/利益关联，在核心论断后一句话点明
- **决策导向** — 信息缺口和决策点要站在读者（仁清）的视角，不是作者视角

## 评论/多人讨论分析（强制）

当文档包含评论区、讨论区、或多人发言时，骨架提取后必须增加以下环节：

### 第一层：动机穿透（每人必做）
不要停在"他问了什么"，必须追问"他**为什么**问这个"：

```
| 他问了什么 | 背后真实质疑 |
|-----------|------------|
| （原文问题） | （推理：他在怀疑/保护/试探什么？） |

**此人的底层逻辑：**
> （1-2句：他的立场、利益、角色决定了他为什么会这样问）
```

### 第二层：集体叠加（强制）
所有人分析完后，**必须回答**：
> "这些人的质疑叠在一起，指向同一个什么问题？"

用树状结构呈现各人质疑如何汇聚到同一个底层担忧。

### 第三层：信号升级（强制）
对每个关键发现，追问：
> "如果这个判断是对的，**最大的后果**是什么？"

不是停在"值得关注"，而是给出后果判断。

## 高级选项

- 若 `user-profile` skill 提供了分析滤镜 → 用滤镜过滤输出重点
- 若 `hypothesis-tracker` skill 有假设库 → 标注哪些假设受此文档影响
- 用户说"展开"某层 → 该层不受条数限制，输出详细分析
